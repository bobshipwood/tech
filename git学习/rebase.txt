我在本地的bob分支下
git rebase dev
git checkout dev
git merge dev



三分支应用情况
c1->c2->c5->c6                             master
           ->c3->c4->c10 	      server
                  ->c8->c9                     client

需求：client 分支合并到master分支，但暂时不想合并server分支
用例  ：--onto选项（选中client分支但不在server分支里的修改，既c8和c9）
在master分支下进行重放：
git rebase --onto master server client
以上命令的意思是取出client分支，找出client分支和server分支的共同祖先之后的修改，然后在master重放一遍

执行命令后变成了
                      master（c6处）
c1->c2->c5->c6->c8`->c9`                            
                             client（c8`处）
           ->c3->c4->c10 	      server

之后执行快速合并client分支
git checkout master
git merge client


然后
git rebase master server（省去切换至server分支，再执行变基的多个步骤）
可以将server分支变基到目标分支上

c1->c2->c5->c6->c8`->c9`->c3->c4->c10     
		     master             server

最后执行
git checkout master
git merge server
